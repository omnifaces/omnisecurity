What is OmniSecurity?

OmniSecurity provides a basic security (login) facility for simple, self-contained web applications with easy of use abstractions on top of Java EE security APIs like JASPIC and JACC.

The focus is on integrating with the Java EE security model and to explore a mechanism that more naturally integrates with JSF than the existing natively supported FORM based authentication options in Java EE's Servlet API.

This project takes advantage of the JASPIC/JASPI/JSR 196 SPI and aims at building a portable SAM (Server Authentication Module/Server Auth Module) that delegates the actual authentication (user name and role retrieval) to user code. It's an implementation of the idea proposed at JAVAEE_SPEC-9.

OmniSecurity is a sub-project of OmniFaces.
It targets Java EE 6 exclusively and is build with JDK 7.
Note that this is still an alpha quality project at the moment.

Installation

Maven coordinates:

<dependency>
    <groupId>org.omnifaces</groupId>
    <artifactId>omnifaces-security</artifactId>
    <version>0.1</version>
</dependency>
